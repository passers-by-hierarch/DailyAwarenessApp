## 1.7 动效规范

### 过渡时长

| 名称 | 值 | 用途 |
|------|-----|------|
| `--duration-fast` | 150ms | 微交互（按钮hover、图标切换） |
| `--duration-normal` | 250ms | 普通过渡（页面切换、展开折叠） |
| `--duration-slow` | 400ms | 大动效（弹窗出现、卡片展开） |

### 缓动函数

```css
--ease-default: cubic-bezier(0.4, 0, 0.2, 1);
--ease-in: cubic-bezier(0.4, 0, 1, 1);
--ease-out: cubic-bezier(0, 0, 0.2, 1);
--ease-bounce: cubic-bezier(0.68, -0.55, 0.265, 1.55);
```

## 2.3 首页交互状态说明

### 正常态

| 状态 | 说明 |
|------|------|
| 有记录 | 时间线列表展示已记录行为，最多显示10条，超出折叠"查看更多" |
| 有事程 | 待办事程展示今日未完成事程，按时间排序，强制事程置顶 |
| 全部完成 | 顶部显示"今日已完成所有事程 ✓"，待办事程区域隐藏 |

### 空态

| 状态 | 说明 |
|------|------|
| 首次使用 | 时间线空态图标 + "还没有记录，点击下方按钮开始" |
| 今日无记录 | 空态图标 + "今天还没记录任何行为" |
| 今日无事程 | 待办事程区域隐藏，或显示"今天没有安排事程" |

### 加载态

| 状态 | 说明 |
|------|------|
| 页面加载 | 顶部状态区骨架屏（一行灰条） |
| 时间线加载 | 3个骨架屏条（高度64px） |
| 事程加载 | 2个骨架屏条（高度72px） |

### 录音态

| 状态 | 说明 |
|------|------|
| 按下按钮 | 按钮缩放0.98，背景变亮，波形动画开始 |
| 正在录音 | 提示文字变为"正在录音..."，麦克风图标变为红色录音指示 |
| 录音结束 | 提示文字变为"识别中..."，波形停止，显示加载spinner |
| 识别成功 | 提示文字变为"已记录：XX"，1秒后恢复默认态 |
| 识别失败 | 提示文字变为"识别失败，请重试"，3秒后恢复默认态 |

### 列表交互

| 操作 | 效果 |
|------|------|
| 时间线项右滑 | 显示"编辑"和"删除"按钮，滑动距离80px触发 |
| 时间线项点击 | 跳转时间线详情页（显示原始语音内容、匹配关系） |
| 事程项左滑 | 显示"完成"、"延后"、"跳过"三个按钮 |
| 事程项点击 | 跳转事程详情页 |
| 强制事程左滑 | 仅显示"完成"和"延后"，无"跳过"按钮 |

## 2.4 首页跳转关系

```
首页
  │
  ├─→ [语音按钮] → 录音流程 → 成功后添加到时间线
  │
  ├─→ [手动输入] → 手动输入弹窗 → 输入文字 → 添加到时间线
  │
  ├─→ [快速事程] → 快速创建事程弹窗
  │                   ├─→ 语音创建 → 编辑/创建事程页
  │                   ├─→ 文字创建 → 编辑/创建事程页
  │                   └─→ 从常用事程选择 → 常用事程列表页
  │
  ├─→ [时间线项] → 时间线详情页
  │                   ├─→ 查看原始语音内容
  │                   ├─→ 查看匹配的事程 → 事程详情页
  │                   ├─→ 编辑记录
  │                   ├─→ 取消匹配
  │                   └─→ 删除记录
  │
  ├─→ [事程项] → 事程详情页
  │                   ├─→ 事程基本信息
  │                   ├─→ 提醒规则配置
  │                   ├─→ 条件触发设置
  │                   ├─→ 必做提醒策略（必做事程）
  │                   ├─→ 编辑事程 → 编辑/创建事程页
  │                   └─→ 删除事程
  │
  └─→ [左滑完成] → 事程标记完成 → 弹出确认 → 更新状态
```

# 第十部分：特殊场景处理

## 10.1 强制日程特殊处理

| 场景 | 处理方式 |
|------|----------|
| 列表中显示 | 左侧红色边框4px，图标为❗，标签显示"强制" |
| 左滑操作 | 仅显示"完成"和"延后"，无"跳过"按钮 |
| 事程详情 | 显示完整的escalating策略配置区域 |
| 超时提醒 | 按escalating策略执行多阶段提醒 |
| 家属通知 | 超时指定时间后自动通知家属 |
| 免打扰时段 | 仅显示通知栏，不播放声音/震动，家属通知仍生效 |

## 10.2 无时间事程处理

| 场景 | 处理方式 |
|------|----------|
| 创建时未指定时间 | 弹出时间推断弹窗 |
| 有历史数据 | 显示建议时间（基于历史平均） |
| 无历史数据 | 显示默认时段选项（晨间/上午/午间/下午/晚间） |
| 用户选择不固定 | 创建为"浮动事程"，无固定时间 |
| 浮动事程触发 | 状态感知：空闲检测、事程间隙、日终兜底 |

---

## 10.3 多轮对话处理

| 场景 | 处理方式 |
|------|----------|
| 用户追问 | AI回答关联前文上下文 |
| 指代模糊 | AI主动澄清："您是指XX还是YY？" |
| 多意图问题 | AI拆解为子查询，汇总回答 |
| 回答来源标注 | 每条回答标注来源记录的时间和类型 |
| 回答中链接 | 关键信息（日期、物品名）可点击跳转 |

## 25.2 数据模型

```typescript
// 语音记录统一模型
interface VoiceRecord {
  id: string
  rawText: string                    // 原始语音文本
  timestamp: string                  // 记录时间
  type: 'item' | 'shopping' | 'behavior' | 'event'
  extractedData: ExtractedData       // 抽取的结构化数据
  matchedAgendaId?: string           // 匹配的事程ID
  status: 'processed' | 'pending'
}

// 1. 物品位置数据
interface ItemLocationData {
  itemName: string                   // 物品名称
  location: string                   // 存放位置
  action: 'place' | 'take' | 'find'  // 放置/拿取/找到
}

// 2. 购物数据
interface ShoppingData {
  items: Array<{
    name: string                     // 商品名称
    quantity: string                 // 数量
    unit: string                     // 单位
  }>
  store?: string                     // 商店（可选）
}

// 3. 行为数据
interface BehaviorData {
  behavior: string                   // 行为名称
  category: string                   // 分类：健康/饮食/运动/生活
  duration?: string                  // 持续时间（可选）
}

// 4. 事件数据
interface EventData {
  event: string                      // 事件名称
  location?: string                  // 地点（可选）
}

type ExtractedData = ItemLocationData | ShoppingData | BehaviorData | EventData
```

## 25.3 4种记录类型识别规则

| 类型 | 关键词特征 | 抽取字段 | 关联去向 |
|------|-----------|----------|----------|
| **物品位置** | "放"、"放在"、"放在了" + 物品名 + 位置 | itemName, location, action | → 物品位置记忆（自动更新） |
| **购物记录** | "买了"、"购买了" + 商品名 + 数量 | items[], store | → 购物记录列表（自动保存） |
| **行为活动** | "正在"、"刚吃完"、"在" + 行为动词 | behavior, category | → 匹配事程 + 行为规律统计 |
| **日常事件** | "拿了"、"回家了"、"出门了" | event, location | → 仅时间线记录 |
