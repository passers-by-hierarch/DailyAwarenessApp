# 生活助手 - 启动指南

> 本指南面向第一次使用的小白用户，请按步骤操作。

---

## 第一步：确认电脑已安装以下软件

启动本项目前，需要电脑上已经装好 **Flutter** 和 **Git**。如果已经装好了，直接跳到第二步。

### 如果没装过 Flutter

1. 打开浏览器，访问这个下载地址：
   ```
   https://storage.flutter-io.cn/flutter_infra_release/releases/stable/windows/flutter_windows_3.24.4-stable.zip
   ```
2. 下载完成后，把压缩包里的 `flutter` 文件夹解压到电脑任意位置（比如 `D:\flutter`）
3. 不需要配置环境变量，脚本会自动找到它

### 如果没装过 Git

1. 打开浏览器，访问：
   ```
   https://git-scm.com/download/win
   ```
2. 下载后双击安装，一直点"下一步"即可
3. 或者直接把绿色免安装版解压到任意位置（比如 `D:\Git`）

> 提示：软件装在哪里都可以，脚本会自己搜索，不需要手动配置环境变量。

---

## 第二步：开启 Windows 开发者模式

Flutter 项目需要开启开发者模式才能正常运行。

**操作方法：**

1. 按键盘上的 `Win + I`（打开设置）
2. 点击左侧 **"隐私和安全性"**
3. 点击右侧 **"开发者选项"**
4. 把 **"开发者模式"** 开关打开
5. 关闭设置窗口

> 只需要开启一次，以后不需要再操作。

---

## 第三步：启动服务

1. 打开项目文件夹，找到 `scirpt` 文件夹
2. 双击里面的 **`start-web-portable.bat`**
3. 会弹出一个黑色窗口，等待执行完成

**正常会看到以下输出：**

```
==========================================
  Daily Awareness Flutter Web Launcher
==========================================

[1/6] Checking Project Structure...
OK: Project valid

[2/6] Checking Git...
OK: Git ready

[3/6] Checking Flutter...
Found Flutter at D:\flutter
OK: Flutter ready

[4/6] Checking Developer Mode...
OK: Developer Mode checked

[5/6] Getting Dependencies...
OK: Dependencies ready

[6/6] Starting Daily Awareness...
Visit: http://localhost:8088
Press Ctrl+C to stop
==========================================
```

4. 等待出现 `Visit: http://localhost:8088`，说明启动成功
5. 打开浏览器，访问 `http://localhost:8088` 即可使用

> 首次启动时，第5步 `[5/6] Getting Dependencies...` 可能需要等待 3-5 分钟，这是正常的，不要关闭窗口。

---

## 第四步：停止服务

用完之后需要停止服务，释放电脑资源。

**操作方法：**

1. 打开项目文件夹的 `scirpt` 文件夹
2. 双击 **`stop-web.bat`**
3. 等待显示 "All services stopped" 即可关闭窗口

> 或者直接在启动服务的黑色窗口里按 `Ctrl + C`，然后按 `Y` 确认退出。

## 常见问题

### 1. 双击启动脚本后窗口一闪就消失了

可能是脚本出错了。重新双击启动，如果还是闪退，查看同目录下的 `startup.log` 文件，里面有详细的错误信息。

### 2. 提示 "Git not found" 或 "Flutter not found"

说明脚本没搜索到。请确认：
- 软件是否已经解压/安装
- 如果装在很深层的路径（比如 `D:\我的文件\工具\flutter`），脚本可能搜不到
- 此时脚本会提示你手动输入安装路径，按提示输入即可

### 3. 提示 "Developer Mode is not enabled"

请按第二步的说明，去系统设置里开启开发者模式。

### 4. 浏览器打不开 http://localhost:8088

- 确认启动窗口还在运行（没有关闭）
- 确认网址输入正确，是 `http://localhost:8088`
- 尝试换 Edge 浏览器打开

### 5. 启动窗口显示 "ERROR: Failed to get dependencies"

可能是网络问题导致依赖下载失败。尝试：
- 检查网络连接
- 重新双击启动脚本再试一次
- 查看 `startup.log` 了解具体错误

## 文件说明

| 文件 | 作用 |
|------|------|
| `scirpt/start-web-portable.bat` | 启动服务（双击运行） |
| `scirpt/stop-web.bat` | 停止服务（双击运行） |
| `scirpt/startup.log` | 启动日志（出问题时查看） |
| `frontend/` | 项目源代码文件夹 |
