# 01 - 项目概述与技术架构

## 1. 项目背景与目标

### 1.1 产品定位

**私人助理（Daily Awareness App）** 是一款面向需要规律性行为管理人群的语音记录与提醒助手。

**核心场景：**
- 老年人/慢性病患者按时服药提醒
- 上班族养成健康生活习惯（喝水、运动、休息）
- 日常事务管理（拿快递、整理房间、预约事项）

**差异化优势：**
1. **语音驱动** - 降低记录门槛，适合不擅长文字输入的用户
2. **智能匹配** - 自动关联行为记录与计划事程，无需手动确认
3. **温和提醒** - 多级提醒强度，避免"一刀切"的强制体验
4. **行为洞察** - 基于数据分析提供个性化建议

### 1.2 项目目标

| 目标维度 | 具体目标 |
|---------|---------|
| 用户规模 | MVP阶段目标1000活跃用户 |
| 核心指标 | 日均语音记录≥5次，事程匹配率≥70% |
| 技术指标 | 语音识别延迟<2秒，提醒准时率≥95% |
| 商业目标 | 6个月内验证核心价值，准备商业化方案 |

### 3.2 状态管理架构（Riverpod）

```dart
// 示例：时间线状态管理

// 1. 定义状态
@immutable
class TimelineState {
  final List<TimelineRecord> records;
  final bool isLoading;
  final String? error;
  
  const TimelineState({
    this.records = [],
    this.isLoading = false,
    this.error,
  });
  
  TimelineState copyWith({
    List<TimelineRecord>? records,
    bool? isLoading,
    String? error,
  }) {
    return TimelineState(
      records: records ?? this.records,
      isLoading: isLoading ?? this.isLoading,
      error: error,
    );
  }
}

// 2. 定义Notifier
class TimelineNotifier extends StateNotifier<TimelineState> {
  final TimelineRepository _repository;
  
  TimelineNotifier(this._repository) : super(const TimelineState());
  
  Future<void> loadTodayRecords() async {
    state = state.copyWith(isLoading: true);
    
    try {
      final records = await _repository.getRecordsByDate(DateTime.now());
      state = state.copyWith(records: records, isLoading: false);
    } catch (e) {
      state = state.copyWith(error: e.toString(), isLoading: false);
    }
  }
  
  Future<void> addRecord(TimelineRecord record) async {
    await _repository.insertRecord(record);
    state = state.copyWith(records: [...state.records, record]);
  }
}

// 3. 定义Provider
final timelineProvider = StateNotifierProvider<TimelineNotifier, TimelineState>((ref) {
  return TimelineNotifier(ref.watch(timelineRepositoryProvider));
});

// 4. 在UI中使用
class TimelinePage extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final timelineState = ref.watch(timelineProvider);
    
    if (timelineState.isLoading) {
      return const LoadingIndicator();
    }
    
    return ListView.builder(
      itemCount: timelineState.records.length,
      itemBuilder: (context, index) {
        return TimelineItem(record: timelineState.records[index]);
      },
    );
  }
}
```

## 7. 性能指标与优化

### 7.1 关键性能指标（KPI）

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
| **语音识别延迟** | < 2秒 | 从录音结束到返回文字的时间 |
| **API响应时间** | < 200ms (P95) | Prometheus监控 |
| **提醒准时率** | ≥ 95% | 提醒触发时间与计划时间的偏差 |
| **事程匹配准确率** | ≥ 85% | 用户反馈修正率 |
| **并发用户数** | 1000+ | 压测验证 |
| **离线可用性** | 90%功能可用 | 本地缓存+离线匹配 |

### 7.2 性能优化策略

| 优化点 | 策略 |
|-------|------|
| **语音识别延迟** | 端侧预处理（降噪、压缩），流式传输，就近节点部署 |
| **API响应** | Redis缓存热点数据，数据库索引优化，连接池 |
| **提醒准时** | 本地定时器+云端双重保障，提前调度 |
| **匹配准确率** | 关键词库优化，时间窗口动态调整，用户反馈学习 |
| **并发能力** | 无状态服务设计，水平扩展，异步处理 |
